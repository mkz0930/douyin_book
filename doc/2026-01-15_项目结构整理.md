# 项目结构整理记录 (2026-01-15)

## 1. 整理背景
随着项目功能增加，根目录下的测试脚本文件（`test_*.py`）越来越多，导致项目结构杂乱。为了保持整洁，决定建立专门的测试目录。

## 2. 变更内容
### 2.1 目录结构调整
-   **创建 `tests/` 目录**：用于存放所有测试相关的代码和资源。
-   **文件迁移**：将以下文件从根目录移动到了 `tests/`：
    -   `test_hf.py`
    -   `test_karaoke_standalone.py`
    -   `test_local_image.py`
    -   `test_pollinations.py`
    -   `test_pollinations.jpg` (测试产物)
    -   `test_quick_validation.py`
    -   `test_search_optimization.py`

### 2.2 Git 配置更新
更新了 `.gitignore`，为测试目录添加了专门的忽略规则，防止测试产生的临时文件污染代码仓库：
```gitignore
# Test Artifacts
tests/__pycache__/
tests/*.jpg
tests/*.mp3
tests/*.mp4
tests/*.txt
!tests/test_*.py
```
这确保了测试脚本本身被 Git 追踪，但运行测试生成的图片、音频、视频等产物会被忽略。

## 3. 验证
-   确认 `tests/` 目录已创建且包含所有测试文件。
-   确认根目录已无杂乱的测试文件。
